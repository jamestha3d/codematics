Data Types

When writing code to give our computer some instructions, we have to give it some data as input. this input can be of different types. Let us look at some data types below

INTEGER

These are whole numbers, negative or positive such as 1 or 2 or 50 or -200, etc. Integer is usually written as int

FLOAT

This is any number with a decimal point in it. for example, 0.7 or 634.81. Float is usually written as float

CHAR

Short for character, This is any single printable character on your keyboard, for example A or B or c or ; or \ etc. NOTE: chars are usually enclosed in single quotes, such as 'A'. Char is usually written as char

STRING

This is a collection of chars, Basically any text on your computer is a string. Examples of strings are "Hello", "Hi", "How are you doing today" or "#$@!?" are all strings. Notice that they are enclosed in quotes.

NOTE: For most programming languages, you can use either single quotes or double quotes for either strings or chars, but a common practice is to use single quotes for chars and double quotes for strings . String is usually written as str

BOOLEAN

This is a value of either True or False. Basically just yes or no. Boolean is usually written as bool

NULL

This usually represents an invalid or non existent item. for example if you try to access a deleted item, your compiler will give you a null response. Null is a very powerful concept in programming which allows us to know when something has gone wrong in our code. Null is usually written as null

NOTE!: Other Data Types worth mentioning such as

  • DOUBLE which is a float but allows higher values than Float, basically a bigger Float
  • LONG which is an integer but allows higher values than integer, Basically a bigger Integer

These data types which we have seen so far are known as Primitive data types

Variables

A variable is a container used to store any of these data types which we have talked about so far. for example, if i have an integer like 15, i can store it in a variable called x. Different programming languages have their way of doing this, but generally this will look something like this age = 15. If i have the string "john", i can also store this in a variable called name and it will look something like this name = "john". A variable is a container that allows us to store our data. all data types can be stored in a variable. this will generally look like variable_name = data . The technical way to say this is that we assign the value of the data to the variable_name. so for age = 15 we say that we assigned the value of 15 to a variable called age. Meanwhile this whole expression where we assign a value to a variable is known as a declaration.

Once declared, variables can be updated in much the same way we declared them. so if we no longer want x to be 15, we can update it to 16 by writing x = 16,

in some programming languages we have to specify what type of data we are storing in the variable at the time of declaration, by writing the data type first. e.g in C programing language, if we want to store the char 'A' in a variable called alphabet. we would do char alphabet = 'A'. This just tells the compiler that alphabet is a variable that stores only chars. More recent programming languages, like javascript or python do not need this specification, and indeed variables can be updated to different types in javascript/python. meaning that a variable can hold a char, and then we can easily update it to a string or a float. For this reason, In JavaScript, our language of choice, we do not have to specify the type of data the variable will hold when declaring the variable(the variable doesn't hold only 1 type of data it can change). We can just do alphabet = 'A'. However JavaScript has 3 optional specifiers that we can put before the variable name. let us look at these

Let

We can specify a variable with the keyword let as follows: let age = 15. It basically just creates a variable called age and assigns the value 15 to it.This is the default option that the computer chooses in the background if we do not specify anything and just write age = 15

Const

Short for constant, we can put const in front of a variable, at the time of declaration. example const age = 15. This makes it impossible for us to change/update age to 16, for example. if we tried to say later in our code age = 16. it would not work and we will get an error. This can be helpful when we are writing programs and we do not need certain variables to change. for example, if we create a variable to store the value of pi. we would not want it to mistakenly change somewhere in our code, and if that is attempted we should get an error and fix the problem

Var
Var is similar in function to let, except that it makes the variable have a global scope instead of local scope. we talk more about this here in scopes.
